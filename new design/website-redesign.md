# MVTT 官网重构提案

> **版本**: v1.0  
> **日期**: 2026-05-31  
> **当前站点**: https://ai.uoyo.net  
> **参考站点**: https://impossibleweb.aura.build/

---

## 1. 现状分析

### 当前页面结构

| 区域 | 内容 | 问题 |
|------|------|------|
| Hero | 标题 + 副标题 + 终端动画 + CTA | 布局偏传统左右分栏，视觉冲击力不足 |
| Features | 3×3 网格 × 9 个特性卡片 | 卡片信息密度均等，无层次递进，用户容易视觉疲劳 |
| Workflow | 5 步垂直时间线 | 代码块样式简陋，步骤之间缺乏连接感 |
| Agents | 6 个角色卡片水平排列 | emoji 图标风格不统一，缺少视觉张力 |
| Stories | 3 个使用场景卡片 | 标签+列表形式过于扁平 |
| Commands | 4 分类命令表 + 终端模拟 | 表格密集，难以快速扫描 |
| CTA / Footer | 安装命令 + GitHub 链接 | 缺乏号召力 |

### 核心问题

1. **视觉层次扁平** — 所有板块权重相同，没有"呼吸感"
2. **Hero 区冲击力弱** — 左文右码的经典布局缺少记忆点
3. **内容密度不均** — Features 区 9 张卡片一次铺开，Commands 区过于表格化
4. **缺少社会证明** — 没有数字指标、用户评价或增长数据
5. **动效缺失** — 页面几乎没有滚动触发动画，缺乏交互感

---

## 2. 重构目标

| 维度 | 目标 |
|------|------|
| **视觉冲击** | 首屏 3 秒内传达核心价值，打造记忆锚点 |
| **信息层次** | 渐进式揭示：概念 → 能力 → 流程 → 角色 → 行动 |
| **交互体验** | 滚动驱动动画 + 微交互，引导用户自然下探 |
| **可信度** | 增加数据指标和真实使用场景佐证 |
| **转化率** | 每个板块结束后都有自然的行动出口 |

## 4. 各区域详细设计

### S0 — Navigation（固定导航栏）

**布局**: 固定顶部，初始透明，滚动后变为毛玻璃（`backdrop-filter: blur`）

```

```

**改进点**:
- 导航项精简为 5 个锚点
- GitHub Star 计数器实时显示（增加信任感）
- `Get Started` 按钮在滚动后出现高亮脉冲

### S1 — Hero（全屏沉浸式首屏）

**参考**: ImpossibleWEB 的全屏 Hero 设计 — 大字体居中、背景视觉特效、状态标签

**布局**: 全屏居中式（非左右分栏），内容垂直堆叠

```
                    ┌──────────────────────────────────────┐
                    │                                      │
                    │     ● Built for Claude Code          │  ← 状态标签（绿色脉冲圆点）
                    │                                      │
                    │    Your AI Dev Team,                  │  ← 主标题，超大字体
                    │       One Command Away.               │     关键词渐变色高亮
                    │                                      │
                    │   29 个 AI 技能驱动的虚拟开发团队       │  ← 副标题，柔和灰色
                    │   分析 → 设计 → 实现 → 评审 → 测试     │
                    │                                      │
                    │  [npx @uoyo/mvtt install]  ← 一键复制  │  ← 安装命令（可点击复制）
                    │                                      │
                    │  [View on GitHub]   [Quick Start →]   │  ← 双 CTA
                    │                                      │
                    └──────────────────────────────────────┘
```

**背景视觉**:
- 深色基底 + 粒子/网格动画（类似矩阵雨效果但更克制）
- 或使用渐变光斑（从左下到右上的蓝紫渐变光晕），呼应 Claude 品牌色
- 底部渐变过渡到下一个板块

**动效**:
- 标题文字逐词淡入 + 轻微上移
- 安装命令区域发光边框动画
- 背景粒子跟随鼠标轻微偏移（视差效果）

**与现有设计的区别**:
- 去掉右侧终端动画（移到下方或 Workflow 区域展示）
- 标题居中，视觉重心更聚焦
- 安装命令作为独立的交互元素，而非隐藏在 CTA 按钮中

## 5. 设计风格与配色方案

### 设计风格

| 维度 | 方案 |
|------|------|
| **基调** | 暗色系科技风（Dark Mode first），参考 ImpossibleWEB 的深色沉浸感 |
| **排版** | 大留白 + 超大标题 + 细腻正文，建立清晰的视觉层次 |
| **卡片** | 微光边框 + 毛玻璃效果（glassmorphism），hover 发光 |
| **图标** | 统一使用线性 SVG 图标（Lucide / Phosphor），弃用 emoji |
| **字体** | 标题: `Inter` 或 `Geist`（700/800 weight）；正文: `Inter`（400）；代码: `JetBrains Mono` |
| **圆角** | 全局 `border-radius: 12px`，按钮 `8px` |
| **间距** | 8px 基础网格系统，板块间距 `120px ~ 160px` |

### 配色方案

#### 主色板

| 角色 | 色值 | 用途 |
|------|------|------|
| **Background** | `#0A0A0F` | 页面主背景 |
| **Surface** | `#12121A` | 卡片/面板背景 |
| **Border** | `rgba(255,255,255,0.06)` | 卡片边框 |
| **Border Hover** | `rgba(99,102,241,0.4)` | hover 发光边框 |
| **Text Primary** | `#F5F5F7` | 标题文字 |
| **Text Secondary** | `#8B8B9E` | 正文/描述文字 |
| **Text Tertiary** | `#4A4A5C` | 辅助信息 |

#### 强调色

| 名称 | 色值 | 用途 |
|------|------|------|
| **Primary** | `#6366F1` (Indigo) | 主 CTA、链接、关键高亮 |
| **Primary Gradient** | `#6366F1 → #8B5CF6` | 按钮渐变、标题关键词 |
| **Accent Cyan** | `#22D3EE` | 次级强调、终端提示符 |
| **Accent Green** | `#10B981` | 成功状态、✓ 标记 |
| **Accent Orange** | `#F59E0B` | 警告、注意项 |

#### 角色专属色（用于 Agent 卡片）

| 角色 | 色值 | 渐变方向 |
|------|------|---------|
| Conductor | `#6366F1` → `#818CF8` | 左→右 |
| Analyst | `#3B82F6` → `#60A5FA` | 左→右 |
| Architect | `#8B5CF6` → `#A78BFA` | 左→右 |
| Developer | `#10B981` → `#34D399` | 左→右 |
| Reviewer | `#F59E0B` → `#FBBF24` | 左→右 |
| Tester | `#22D3EE` → `#67E8F9` | 左→右 |

#### 渐变效果

## 6. 动效规范

### 全局原则

- 所有动画采用 `ease-out` 缓动，时长 `300ms ~ 600ms`
- 滚动触发动画使用 `Intersection Observer`，阈值 `0.2`
- 避免过度动画，保持克制优雅

### 各区域动效清单

| 区域 | 动效 | 触发条件 |
|------|------|---------|
| Hero 标题 | 逐词淡入 + 上移 8px | 页面加载 |
| Hero 背景 | 粒子/光斑缓慢飘动 | 持续 |
| Hero 命令框 | 边框发光脉冲 | 持续（2s 循环） |
| Social Proof 数字 | countUp (0 → N) | 进入视口 |
| Feature 卡片 | 从下方淡入，stagger 100ms | 进入视口 |
| Feature 卡片 hover | 上浮 4px + 边框发光 | hover |
| Workflow 管道 | 线条从左到右点亮 | 进入视口 |
| Workflow 终端 | 打字机逐行输出 | 步骤切换时 |
| Agent 卡片 | 水平滑入 | 进入视口 |
| Agent 卡片 hover | 翻转显示背面 | hover |
| Story 卡片 | 左右交替滑入 | 进入视口 |
| Story ✅ 标记 | 弹跳出现 | 卡片动画完成后 |
| CTA 命令框 | 脉冲发光 | 持续 |
| 导航栏 | 透明→毛玻璃 | 滚动 > 100px |
